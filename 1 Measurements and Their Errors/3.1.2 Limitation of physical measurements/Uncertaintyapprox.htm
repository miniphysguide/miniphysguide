How we approximate uncertainty in multiplication/division :)


Shalom! Well done for being curious haha.

When we add the percentage uncertainties in multiplication/division, this is in fact an approximation because it's much less hassle and usually works out to be quite similar to what the actual uncertainty would be.

We can see this by figuring out an expression for the actual uncertainty (the range of possible values) and comparing it to what we would get by adding the percentage uncertainties.

Uncertainty: multiplication

Consider multiplying two positive values: (k ± b) and (m ± a).

Note: Remember that ± b and ± a both represent absolute uncertainty.

You can try do this yourself first: work out the max. value of the product and then the min. value.

See if you can represent the result as a value with an uncertainty using this information.

Then work out the algebraic result of using the 'adding percentage uncertainties' method and compare the two.


Product of two positive values. (k ± b) × (m ± a)

          Max value = (k + b) × (m + a)
                                = km + ka + bm + ba

             Min value = (k - b) × (m - a)
                                 = km - ka - bm + ba
                                 = km + ba - (ka + bm)

     ∴ we can write the result as (km + ba) ± (ka + bm).

  As a % uncertainty, it'd look like this:

                          (km + ba) ± ( ka + bm⁄km + ba [× 100] ) %

  ^ this is the actual answer we should get.


Instead, by using the 'adding percentage uncertainties' rule, we get this:

Product of two positive integers. (k ± b) × (m ± a)

Rewrite as % uncertainties:

                (k ± ( b⁄k [× 100] ) %) × (m ± ( a⁄m [× 100] ) %)

ans = km ± (sum of % uncertainties)%
        = km ± (( b⁄k + a⁄m) [× 100] ) %
        = km ± ( ka + bm⁄km [× 100] ) %


Comparing the two...

(km + ba) ± ( ka + bm⁄km + ba [× 100] ) %       ←actual
vs
km ± ( ka + bm⁄km [× 100] ) %            ←approximation by adding %                                                                      uncertainties

...we can see that the results are very similar.


We are able to use this approximation because what it basically does is assume that ba is small enough to be insignificant compared to km (so we can treat 'km + ba' to be similar to just 'km' by itself). This is usually true because b and a are generally much smaller values than k and m. Multiplying b and a produces an even smaller value because they both tend to be values less than 1- the opposite is true for multiplying k and m.

Uncertainty: division

Now, consider dividing (k ± b) by (m ± a) where they are both positive values.

After seeing how we came to a conclusion for multiplication together, it should be clear how we approach doing the same for division.

Grab a pencil and paper and try to figure this one out for yourself before checking with mine!

Result from dividing two positive values. (k ± b)⁄ (m ± a)

            Max value = (k + b) ⁄ (m - a)
                                 = (k + b)(m + a) ⁄ (m - a)(m + a)
                                 = (km + ka + bm + ba) ⁄ (m2 - a2)
                                 = (km + ba) ⁄ (m2 - a2) + (ka + bm) ⁄ (m2 - a2)

              Min value = (k - b) ⁄ (m + a)
                                 = (k - b)(m - a) ⁄ (m + a)(m - a)
                                 = (km - ka - bm + ba) ⁄ (m2 - a2)
                                 = (km + ba) ⁄ (m2 - a2) - (ka + bm) ⁄ (m2 - a2)

      ∴ we can write the result as (km + ba) ⁄ (m2 - a2) ± (ka + bm) ⁄ (m2 - a2).

  As a % uncertainty, it'd look like this:

(km + ba)⁄(m2 - a2) ± ( ( (ka + bm)⁄(m2 - a2) ÷ (km + ba)⁄(m2 - a2) ) [× 100] ) %

= (km + ba)⁄(m2 - a2) ± ( ( (ka + bm)⁄(km + ba) ) [× 100] )%

  ^ you guessed it, this is what the actual answer should be!


Now, by using the 'adding percentage uncertainties' method, we get this:

Result from dividing two positive values. (k ± b)⁄ (m ± a)

Rewrite as % uncertainties:

(k ± ( b⁄k [× 100] ) %) ⁄ (m ± ( a⁄m [× 100] ) %)

ans = k⁄m ± (sum of % uncertainties)%
        = k⁄m ± (( b⁄k + a⁄m) [× 100] ) %
        = k⁄m ± ( ka + bm⁄km [× 100] ) %


Comparing the two...

(km + ba)⁄(m2 - a2) ± ( ( (ka + bm)⁄(km + ba) ) [× 100] ) % ← actual
vs
k⁄m ± ( ka + bm⁄km [× 100] ) %     ← approximation by adding %                                                                              uncertainties

...again, the results are similar(ish).

Applying the same logic as for multiplication, we can assume that a and b are small enough compared to k and m so that we can just treat km + ba as km and we can also treat m2 - a2 as m2. This allows us to use the approximation method.


Pretty nifty, huh? :D
